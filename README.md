# Aplicación Dinero Comprometido

Estructura pública y genérica de una aplicación de gestión financiera personal, preparada para crecer hacia herramientas de operación para pequeños negocios.

## Estado del proyecto

El proyecto se encuentra en definición funcional y visual previa a la construcción. El objetivo inmediato es alcanzar un prototipo personal funcional de nivel 3. El módulo de negocios permanece documentado para evitar retrabajo, pero su implementación se realizará después.

## Idea central

La interfaz contrasta dos cantidades:

- **Dinero comprometido:** recursos reservados para obligaciones, gastos esenciales, pagos y objetivos.
- **Dinero libre:** recursos realmente utilizables después de considerar los compromisos.

## Principios

- Ingresos fijos o variables.
- Registro preciso de movimientos y correcciones trazables.
- Tarjetas de débito, crédito, efectivo, inversiones y otras cuentas.
- Calendario real para vencimientos, pagos e ingresos.
- Comentarios automáticos mediante reglas, sin IA integrada.
- Exportación para análisis externo en formatos interoperables.
- Separación estricta entre finanzas personales y de negocios.
- Rendimiento fluido en dispositivos de recursos limitados.

## Estructura

```text
docs/       Alcance, arquitectura, ruta y método de construcción
schemas/    Contratos de datos públicos y sin información real
src/        Organización prevista del código a partir de F3
tests/      Estrategia prevista de pruebas
```

No se incluyen saldos, instituciones financieras, documentos fiscales, archivos empresariales ni datos personales reales.
